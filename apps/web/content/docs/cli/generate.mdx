## What it does

Reads all migration files from `./damian/.migrations/`, runs them against an in-memory database, and produces three output files:

| Output | Purpose |
|--------|---------|
| `./damian/.generated/tables.ts` | TypeScript table definitions used by the query builder |
| `./damian/.generated/typings.ts` | Re-exports custom typings from `./damian/typings.ts` (or an empty default) |
| `./damian/db.sql` | A clean SQL dump of your current schema (for version control / review) |

## devDumpIgnore

The `devDumpIgnore` option in `damian.config.ts` lets you exclude specific migration files from the `db.sql` dump while still including them in the `tables.ts` generation. This is useful for cleaning up the schema dump by omitting seed data or other non-structural migrations.

```ts title="damian.config.ts"
import { config } from 'damian'

export default config({
    driver: 'postgres',
    url: process.env.DATABASE_URL,
    devDumpIgnore: ['20240101000000_seed_dev_data.sql'],
})
```
